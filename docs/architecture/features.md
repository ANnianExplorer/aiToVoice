## 1. 用户认证模块

### 功能清单

| 功能 | 说明 |
|------|------|
| 注册 | 用户名 + 邮箱 + 密码，密码 BCrypt 加密 |
| 登录 | 用户名 + 密码，返回 JWT Token |
| Token 刷新 | 7 天有效期的 Refresh Token |
| 个人资料 | 昵称、头像、简介 |
| 用户设置 | 主题、语言、音频偏好等 |

### 用户流程

```
注册 → 登录 → 获取 Token → 前端存储 Token → 后续请求携带 Token
                                    ↓
                            Token 过期 → 用 Refresh Token 刷新
                                    ↓
                            Refresh 也过期 → 重新登录
```

### 实现要点

- 密码使用 BCrypt 加密存储
- JWT Token 有效期 24 小时，Refresh Token 7 天
- `JwtAuthenticationFilter` 拦截所有请求，验证 Token
- `SecurityConfig` 配置公开接口（/api/auth/**, GET /api/songs/**）

## 2. 音乐播放模块

### 功能清单

| 功能 | 说明 |
|------|------|
| 播放/暂停 | 控制音频播放状态 |
| 上一首/下一首 | 切换歌曲 |
| 进度条 | 显示/拖动播放进度 |
| 音量控制 | 调节音量，静音 |
| 播放模式 | 顺序播放、随机播放、单曲循环 |
| 播放列表 | 当前播放队列 |

### 用户流程

```
点击歌曲 → 加入播放列表 → Howler.js 加载音频 → 开始播放
                                    ↓
                        进度条实时更新 ← requestAnimationFrame
                                    ↓
                        播放结束 → 根据模式播放下一首
```

### 实现要点

- `playerStore` (Zustand) 管理播放状态
- `useAudio` Hook 桥接 Zustand 和 Howler.js
- 音频文件通过 `/api/files/audio/{id}` 流式传输
- `requestAnimationFrame` 实现平滑进度更新

## 5. 语音录制与分析模块

### 功能清单

| 功能 | 说明 |
|------|------|
| 录音 | 录制用户演唱 |
| 音高分析 | TarsosDSP 检测音高曲线 |
| 评分 | 综合音准、稳定性评分 |
| 与原唱对比 | 对比用户录音和原唱 |
| 练习任务库 | 预设的声乐练习任务 |
| 进度追踪 | 记录练习次数、最佳分数 |

### 分析流程

```
录音上传 → FileStorageService 存储
    ↓
PitchAnalyzer 分析音高 (TarsosDSP YIN 算法)
    ↓
ScoreCalculator 计算综合评分
    ↓
生成反馈建议（基于分数区间）
    ↓
保存分析结果到 voice_records
```

### 评分算法

```java
综合分 = 音准分 × 0.6 + 节奏分 × 0.4
音准分 = 100 - √(音高方差)
稳定性 = 100 - √(音高方差)
```

### 练习任务类型

| 类型 | 说明 | 难度 |
|------|------|------|
| BREATH | 呼吸控制练习 | 1-3 |
| PITCH | 音准练习 | 1-5 |
| RHYTHM | 节奏感练习 | 2-4 |
| VIBRATO | 颤音练习 | 3-5 |

## 6. AI 音乐老师模块

### 功能清单

| 功能 | 说明 |
|------|------|
| 声乐教练对话 | 上传录音，AI 分析发声 |
| 音乐助手 | 音乐知识问答 |
| 会话管理 | 创建、查看历史会话 |
| 练习计划 | AI 生成个性化练习计划 |

### 对话流程

```
用户创建会话（选择类型：声乐教练/音乐助手）
    ↓
发送消息 → AiService 构建对话上下文
    ↓
AiClient 调用外部 API（OpenAI/Claude）
    ↓
保存 AI 回复 → 返回给前端
```

### System Prompt 策略

- **声乐教练模式**: 专业声乐教练角色，分析发声、给出练习建议
- **音乐助手模式**: 音乐知识丰富的助手，推荐歌曲、讲解理论

## 7. 个性化推荐模块

### 推荐算法

| 算法 | 说明 | 权重 |
|------|------|------|
| 协同过滤 | 基于相似用户的听歌行为 | 40% |
| 内容推荐 | 基于歌曲流派、标签相似度 | 40% |
| 热门趋势 | 基于全局播放量 | 20% |

### 推荐场景

| 场景 | 说明 |
|------|------|
| 每日推荐 | 每天 30 首个性化推荐 |
| 私人 FM | 连续播放，随机推荐 |
| 相似歌曲 | 基于当前歌曲推荐同类 |
| 推荐反馈 | 用户喜欢/不喜欢调整推荐权重 |

## 8. 歌词功能模块

### 功能清单

| 功能 | 说明 |
|------|------|
| LRC 解析 | 解析 LRC 格式歌词（时间戳 + 文本） |

| 歌词搜索 | 从在线源获取歌词 |
| 桌面歌词 | Electron 独立悬浮窗口（待实现） |

### LRC 格式

```
[00:01.00]第一行歌词
[00:05.50]第二行歌词
[00:10.20]第三行歌词
```

解析为 `Map<Long, String>`（毫秒 → 歌词文本）
